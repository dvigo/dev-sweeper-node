# Contributing to dev-sweeper

First off, thanks for taking the time to contribute! 🎉

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue with:
- Your operating system and Node.js version.
- Steps to reproduce the bug.
- The expected vs actual behavior.

### Suggesting Enhancements
We are always open to new ideas! If you have a suggestion for a new feature (e.g. supporting a new ecosystem dependency folder):
- Create an issue explaining the feature and why it would be useful.
- Wait for a maintainer to discuss the approach before starting to write code.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Make your changes in TypeScript.
3. Run `npm run build` to ensure the project compiles without errors.
4. Test your changes locally by running `npm start`.
5. Open a Pull Request!

## Development Setup
```bash
npm install
npm run dev # Runs the CLI directly via tsx for rapid testing
```
